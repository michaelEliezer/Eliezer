Section 01
01

Inspection Standard

Every screen is judged
before it is shown.

01

Remove the decorative

If it doesn't carry information or guide attention, it doesn't belong. Every element must justify its presence.

02

Stress-test the layout

Shorter viewports. Longer text. Missing images. Weak content. The layout must hold under pressure, not just in the mockup.

03

Make motion accountable

Every animation must clarify attention or improve pacing. If it exists because it looks cool, it gets cut.

04

Ship only what survives

The final screen is the one that passed every test. Not the one that looked best in the first draft.

Section 04
04

Proof of Craft

What gets inspected,
not displayed.

Load Behavior Target

First paint under 1.5s. No layout shift. Fonts staged. Scripts deferred.

The page must arrive before the user notices it loading. Measured, not assumed.

Layout Resilience Target

320px to 2560px. Content reflows without overflow. No horizontal scroll.

The grid must hold under pressure. Tested, not just reviewed in devtools.

Motion Restraint Pending Audit

Every animation has a reason. Reduced-motion respected.

Movement must serve the reading order. No motion for motion's sake.

Accessibility Pass Target

Semantic HTML. ARIA where needed. Keyboard navigable. Focus visible.

Not an afterthought. Inspected before every ship.

Code Structure Pending Audit

Maintainable class names. No dead CSS. Organized files.

The codebase must read like documentation. Audited, not just working.
